# «Алхимия красоты» *(рабочее название)*

Проект Сергея Васильева для Марины Баяндиной — курс о практической красоте: **кожа, волосы, ногти**. Полный бриф и все ⚠️ открытые вопросы — в [`CONTEXT.md`](./CONTEXT.md).

## Что это за продукт

Тот же фундамент антивозрастной медицины, что и в «Алхимии молодости: 7 ступеней», но продукт **не про здоровье целиком, а только про красоту** — и только про практику. Ступень 07 «Сияние» из «Алхимии молодости», развёрнутая в самостоятельный курс.

Ключевая мысль всего копирайтинга: **красота — это биология, а не баночка.** Каждый блок заканчивается конкретным действием, а не пониманием.

## Состояние

- **Стадия:** собран бриф, весь контент в Markdown и **три варианта оформления — тремя самостоятельными сайтами**: [A «Клиника»](https://razovik-hub.github.io/krasota-a/), [B «Глянец»](https://razovik-hub.github.io/krasota-b/), [C «Тактильный нюд»](https://razovik-hub.github.io/krasota-c/). Выбирает Марина. Два невыбранных потом удаляются вместе с `src/variants/`, страницей выбора `index.astro` и скриптом сборки.
- **Репозитории:** исходник — `razovik-hub/krasota` (деплоит хаб через Actions на push в `main`). Сайты вариантов — `razovik-hub/krasota-{a,b,c}`, туда кладётся готовая статика командой `npm run publish:variants`. **Правки контента — только в исходнике**, в репозитории вариантов ничего руками не менять: следующая публикация всё перезапишет force-пушем.
- **Ассистент:** на первом этапе **не делаем**. CTA ведут на якорь `#cta` — ⚠️ куда падает заявка, ещё не решено.
- **Индексация:** во всех страницах стоит `noindex, nofollow` — это черновик на согласовании. Снять перед запуском.

## Стек

- **Astro** → статика. Контент **строго в Markdown** — `src/content/page/*.md`, `blocks/*.md`, `experts/*.md`, `tariffs/*.md`. **Не зашивать тексты в `.astro`/CSS** — включая `<title>`, он берётся из `hero.md`. Схема коллекций — [`CONTENT-SCHEMA.md`](./CONTENT-SCHEMA.md), из неё один в один пишется `src/content/config.ts`.
- **Вёрстка вариантов** — `src/variants/{A,B,C}.astro`, по файлу на направление. Данные все трое берут через `src/lib/content.ts` — **один загрузчик на всех**, чтобы контент физически не мог разъехаться между вариантами.
- Отличие от Molodost: коллекция программы называется `blocks` (не `steps`), и **экспертов несколько** — отдельная коллекция `experts`, а не синглтон.
- Деплой — GitHub Actions на push в `main`, workflow копируется из Molodost (`DEPLOY=ghpages` → `base=/<name>`).

## Стиль ⚠️ — не выбран

Решено, что визуальный язык **совсем другой**, не как у «Алхимии молодости» (Cormorant + золото + изумруд). Три направления — A «Клиника» / B «Глянец» / C «Тактильный нюд» — описаны в `CONTEXT.md`. После выбора пишем `STYLE-GUIDE.md`.

До выбора **не начинать вёрстку** и не заводить `global.css`.

## Тон копи

Живой русский, **без канцелярита** — Сергей чувствителен к сухости. «Системная программа о том, как…» — нельзя. Premium ≠ казённо.

Что здесь работает особенно хорошо и чего нет в «Алхимии молодости»: можно быть **конкретной и слегка дерзкой**. «Полка из 5 средств вместо 25», «ногти — самый честный анализ, который всегда с собой», «что даёт результат, а что даёт только чек». Это продукт про практику — язык тоже должен быть практичным.

Чего избегать: обещаний вечной молодости, медицинских гарантий, слова «омоложение» в значении процедуры.
